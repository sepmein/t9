API总体介绍

Kokiya.com使用nodejs作为服务器端语言，mongodb作为数据库。

作为一个以API为核心搭建的应用，kokiya的前端尽量都使用API获取数据。为将来扩展客户端做好准备。

目前，API的获取并未增加限制，您可以随意调用。当然希望您“友善”地使用。这时候，感激涕零是唯一的形容词。

作为现在网络API几乎的标准，本api同其他应用一样:遵循restful原则，并且使用JSON作为数据格式。

状态代码参照使用nodejs的格式，分为5个等级，代表api的稳定性。越高越稳定。详细可参考:nodejs官方文档

Posts API

url:"/api/posts"

状态: 1


数据格式:

            {
              _id: ObjectId       //发表者id
              uid: ObjectId       //发表者id
              author: String,     //发表者
              content: String,    //内容
              date: {             //日期，格式ISOdata，可以使用new Date(date)来处理。
                type: Date,
              default:
                Date.now
              },
              meta: {             //元信息
                plus: Number,     //顶
                minus: Number,    //踩
                star: Number      //星标
              },
              comments: [Comment] //评论，格式：数组，详见comment部分
            }

comment:

            {
              author: String,     //评论者
              content: String,    //内容
              date: {             //日期，同上
                type: Date,
              default:
                Date.now
              }
            }

方法:get

结果:获取所有posts

示例:浏览器输入kokiya.no.de/api/posts，返回一串JSON格式数组


方法:post

content字段必须提供。若已建立session，uid以及author可以不提供，服务器会自动获取该两条字段。若未建立字段，可以在post.body中提供

结果:以上述格式插入一条post

Post Comment API

url:"/api/posts/comment"

状态：2


数据格式：

            {
              pid : pid     //post id
              content : String  //评论内容   
              user : String //optional发表用户，由浏览器客户端发送可以不用提供，session中已包含
              uid : uid     //optional，同上
              date : date   //optional,发表日期，不提供可由服务器自动生成。
            }

方法：post

说明：pid是标识被评论文章的重要纽带。必须填写。

结果：若插入成功则返回一个标识成功的状态码

            {ok:1}

若失败则返回一个标识失败的状态码，并附加一个错误代码:

            {
              {ok:0},
              {err:error}
            }

LifeTags API

url:"/api/lifetags"

状态：1


数据格式：

            {
              title: {
                type: String,
                required: true,
                trim: true
              },
              content: String,
              date: {
                type: Date,
                default: new Date
              },
              location: {
                x: Number,
                y: Number
              },
              tags: [Tag]
              users: [
                  uid: {
                    type: ObjectId,
                    ref: 'users',
                    required: true
                  },
                  ltid: {
                    type: ObjectId,
                    ref: 'lifetags',
                    required: true
                  },
                  type: {
                    type: String,
                    enum: ['participant', 'initiator']
                  },
                  privacy: {
                    type: Number, 
                    min: 0,
                    max: 5
                  },
                  importance: {
                    type: Number,
                    min: 0,
                    max: 999
                  },
                  happiness: {
                    type: Number,
                    min: -5,
                    max: 5
                  }
              ],[],[],[],[]..
            }

Users API

url:"/api/users"

状态: 1


数据格式：

            {
              "_id" : id
              , "user" : String   //用户名
            }

方法:get

结果:获取所有用户名

示例:浏览器输入kokiya.no.de/api/posts，返回一串JSON格式数组

Info API

url:"/api/serverInfo"

状态: 4


数据格式：

            {
              "upTime" : Number     //服务器上线时间，单位：秒

方法:get
